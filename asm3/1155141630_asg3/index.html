Challenge Requirements

Query No

It is implemented by cookie, with maxAge 300,000.
In every successful GET requests, the query no follows with the mongoose results, splitted by a comma.
For example, [],{"query_no":3}.
The failed requests are also counted but not displayed.

GraphQL

This part uses GraphQL to fetch the information of events and locations.
It is implemented by Apollo Server and tested by Apollo sandbox.
The sandbox is in https://studio.apollographql.com/sandbox/explorer.
My graphql API is in http://localhost:3000/graphql.
The following is the tested query operations.

Operations

query Query { event(id:1){ eventId name loc { locId name } quota } }

query Query { events{ eventId name loc { locId name } quota } }

query Query{ location(id:1) { locId name quota } }

query Query{ locations { locId name quota } }

My Custom Requests

  • /create: create two location datas,
    [{"locId":1,"name":"SHB924","quota":10},{"locId":2,"name":"ERB LT","quota":5}].
  • /clear: clear all locations and events.
  • /forms.html: forms for Q2, 3, 8.
  • /ev4q8/eventId: GET request for Q8 (used in froms.html).
